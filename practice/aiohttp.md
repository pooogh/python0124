# Асинхронные HTTP-запросы (`aiohttp`)

---

## Как работает HTTP

Взаимодействие в интернете строится на модели **Клиент-Сервер**. 
* **Клиент** (ваш Python-скрипт) отправляет **Запрос** (Request).
* **Сервер** (например, httpbin.org) обрабатывает его и возвращает **Ответ** (Response).

Любой запрос состоит из трех главных частей:
1. **Метод:** Что мы хотим сделать? 
   * `GET` - дай мне данные (скачать страницу, получить профиль).
   * `POST` - возьми мои данные и создай что-то новое (регистрация, отправка сообщения).
2. **URL:** Адрес, куда мы обращаемся (например, `https://api.site.com/users`).
3. **Заголовки (Headers):** Служебная информация, прикрепленная к запросу (кто мы такие, какой браузер используем, есть ли у нас пароль).

У ответа сервера всегда есть **Статус-код**:
* `200 OK` - все отлично, запрос выполнен.
* `400 Bad Request` - вы отправили кривые данные (ошибка на стороне клиента).
* `401 Unauthorized` - вы не авторизованы (нет логина/пароля).
* `404 Not Found` - такого адреса не существует.
* `500 Internal Server Error` - сервер сломался (ошибка на стороне бэкенда).

## Задача 1: Передача параметров в URL (Query Parameters)

**Теоретическая справка**
Часто при GET-запросах нам нужно отфильтровать данные. Например, найти товары дешевле 1000 рублей или открыть 5-ю страницу каталога. В HTTP это делается через параметры URL. Они дописываются в конец адреса после знака вопроса: `https://site.com/search?price=1000&page=5`.
Чтобы не клеить эту строку вручную (что чревато ошибками), современные библиотеки вроде `aiohttp` позволяют передать параметры в виде обычного Python-словаря.

**Условия**
- Напишите корутину `fetch_with_params()`.
- Сделайте асинхронный GET-запрос к `https://httpbin.org/get`.
- Передайте параметры запроса (`key="value"`, `another_key="another_value"`) с помощью словаря.
- Выведите из полученного JSON-ответа ключ `args` (в нем сервер покажет, какие параметры он успешно прочитал).

**Конструкции**
- Аргумент `params=...` в методе `session.get()`

**Пример вывода:**
```text
Полученные параметры сервером:
{'another_key': 'another_value', 'key': 'value'}
```

## Задача 2: Отправка данных формы (Form Data)

**Теоретическая справка**
Когда пользователь заполняет форму на сайте (например, форму входа) и нажимает "Отправить", браузер отправляет POST-запрос. Исторически такие данные передаются не в виде JSON, а в специальном формате `application/x-www-form-urlencoded`. Нам нужно уметь имитировать такое поведение браузера из нашего Python-кода, чтобы авторизовываться на старых сайтах или работать со специфическими API.

**Условия**
- Напишите корутину `submit_form()`.
- Сделайте асинхронный POST-запрос к `https://httpbin.org/post`.
- Передайте словарь с данными `{"username": "admin", "password": "secret"}` в качестве данных формы.
- Распечатайте ключ `form` из JSON-ответа сервера.

**Конструкции**
- Для отправки формы используется аргумент `data=...` в методе `session.post()` (если бы мы отправляли JSON, мы бы использовали аргумент `json=...`).

**Пример вывода:**
```text
Сервер прочитал данные формы:
{'password': 'secret', 'username': 'admin'}
```

## Задача 3: Базовая авторизация (Basic Auth)

**Теоретическая справка**
Самый простой (и самый старый) способ защиты веб-страниц - базовая авторизация. Логин и пароль склеиваются через двоеточие (`admin:secret`), кодируются в формат **Base64** и отправляются в служебном заголовке `Authorization`. Библиотека `aiohttp` делает всю эту черновую работу автоматически.

**Условия**
- Напишите корутину `login_basic()`.
- URL для проверки: `https://httpbin.org/basic-auth/Aladdin/opensesame`. (Этот адрес настроен так, что пустит только пользователя Aladdin с паролем opensesame, он не отображает реальной картины (камон, никто не педелает пароли в адресной строке), но поможет понять принцип).
- Сделайте GET-запрос, передав данные для авторизации.
- Если авторизация успешна, распечатайте статус-код (он должен быть 200) и сообщение об успехе. Если пароль неверный, сервер вернет код 401. Протестируйте оба варианта передачи.

**Конструкции**
- Специальный объект `aiohttp.BasicAuth('логин', 'пароль')`
- Аргумент `auth=...` в методе `session.get()`

**Пример вывода:**
```text
Статус ответа: 200
Авторизация пройдена: {'authenticated': True, 'user': 'Aladdin'}
```

## Задача 4: Работа с Cookies

**Теоретическая справка**
Сам по себе протокол HTTP не помнит пользователя. Чтобы сервер узнавал вас после авторизации, используются куки (Cookies) - маленькие текстовые метки. Вы отправляете куку вместе с каждым запросом, и сервер понимает: "Ага, это тот самый пользователь".

**Условия**
- Напишите корутину `send_cookies()`.
- Сделайте GET-запрос к `https://httpbin.org/cookies`.
- Передайте словарем две куки: `name` со значением `user` и `secret` со значением `secret_hash`.
- Распечатайте ответ сервера, чтобы убедиться, что он успешно распознал ваши куки.

**Конструкции**
- Аргумент `cookies=...` в методе `session.get()`

**Пример вывода:**
```text
Отправленные куки:
{'cookies': {'name': 'user', 'secret': 'secret_hash'}}
```

## Задача 5: Чтение потока данных (Chunked Stream)

**Теоретическая справка**
Обычно сервер ждет, пока сформирует ответ целиком, и только потом отправляет его вам. Но что если данных слишком много (видео, бесконечные логи)? Сервер может отдавать ответ "кусками" (chunks) по мере их готовности. Это называется Streaming. В Python мы можем обрабатывать такие куски по очереди, используя асинхронный цикл, не дожидаясь закрытия соединения.

**Условия**
- Напишите корутину `read_stream()`.
- Сделайте GET-запрос к `https://httpbin.org/stream/3`. (Этот адрес сымитирует стрим и отдаст 3 отдельных куска данных с небольшой задержкой).
- Используйте асинхронный цикл для чтения ответа построчно.
- Выводите в консоль каждый полученный фрагмент.

**Конструкции**
- Асинхронный цикл: `async for line in response.content:`
- Полученные байты нужно будет превратить в обычную строку с помощью `line.decode('utf-8')`.

**Пример вывода:**
```text
Получен чанк: {"url": "[https://httpbin.org/stream/3](https://httpbin.org/stream/3)", "args": {}, ...}
Получен чанк: {"url": "[https://httpbin.org/stream/3](https://httpbin.org/stream/3)", "args": {}, ...}
Получен чанк: {"url": "[https://httpbin.org/stream/3](https://httpbin.org/stream/3)", "args": {}, ...}
Поток завершен.
```

## Задача 6: Обход проверки SSL-сертификата (HTTPS)

**Теоретическая справка**
Буква "S" в протоколе HTTPS означает "Secure" (безопасный). Данные шифруются с помощью SSL-сертификатов. Библиотека `aiohttp` строго следит за безопасностью: если сертификат сайта просрочен или самоподписной (часто бывает на корпоративных серверах при разработке), она выбросит ошибку и запретит подключение. Иногда (исключительно для тестов!) нам нужно принудительно отключить эту проверку.

**Условия**
- URL для тестирования ошибки: `https://self-signed.badssl.com/`. (Это специальный тестовый стенд с кривым сертификатом).
- Напишите корутину `bypass_ssl()`.
- Сделайте GET-запрос к этому URL, специально отключив проверку сертификатов, иначе запрос упадет.
- Используйте блок `try...except asyncio.TimeoutError`, так как тестовые серверы иногда зависают от нагрузки.
- Распечатайте статус-код ответа.

**Конструкции**
- Аргумент `ssl=False` прямо в методе `session.get(...)`

**Пример вывода:**
```text
Запрос к сайту с кривым сертификатом выполнен (проверка SSL отключена).
Статус: 200
```
