# Интеграция HTTP-запросов и Файловой системы

## Преобразование данных (JSON)

Когда мы получаем ответ от сервера через `await response.json()`, `aiohttp` автоматически превращает текстовый JSON в удобный **словарь** (или список) Python. 
Но когда мы хотим сохранить этот словарь в текстовый файл, нам нужно сделать обратное действие.

Для этого используется встроенная библиотека `json`:
* `json.dump(data, file)` - берет Python-словарь и красиво записывает его в открытый файл.
* `json.load(file)` - читает текстовый файл и превращает его обратно в Python-словарь.

## Задача 1: Экспорт данных (GET -> JSON файл)

Вы пишете парсер. Вам нужно скачать список пользователей из сторонней системы и сохранить его к себе на компьютер для дальнейшего анализа.

**Условия**
- Импортируйте встроенный модуль `json`.
- Напишите корутину `export_users()`.
- Сделайте `GET` запрос к `https://jsonplaceholder.typicode.com/users`.
- Получите данные в виде списка словарей.
- Откройте (или создайте) локальный файл `users_export.json` в режиме записи (`"w"`). *Рекомендуется использовать параметр `encoding="utf-8"`.*
- Запишите полученные данные в этот файл с помощью `json.dump()`. Для красоты добавьте параметр `indent=4`.
- Выведите в консоль сообщение об успешном сохранении.

**Конструкции**
```python
with open("filename.json", "w", encoding="utf-8") as f:
    json.dump(ваши_данные, f, indent=4, ensure_ascii=False)
```

**Пример вывода:**
```text
Данные 10 пользователей успешно сохранены в файл users_export.json!
```

## Задача 2: Импорт и модификация данных (Файл -> POST)

У вас есть файл с выгрузкой данных. Вам нужно прочитать его, немного изменить информацию и отправить на другой сервер для создания новой записи.

**Условия**
- Убедитесь, что у вас есть файл `users_export.json` из предыдущей задачи.
- Напишите корутину `import_and_send()`.
- Откройте файл в режиме чтения (`"r"`) и загрузите данные в переменную с помощью `json.load()`.
- Возьмите **первого** пользователя из списка. Измените его имя (ключ `name`) на любое другое (например, "Иван Иванов"). Удалите ключ `id`, так как при создании новой записи сервер сам назначит новый ID.
- Сделайте `POST` запрос к `https://jsonplaceholder.typicode.com/users`, передав измененный словарь в параметр `json`.
- Распечатайте ответ сервера, чтобы убедиться, что запись создана (сервер должен вернуть словарь с новым `id: 11`).

**Пример вывода:**
```text
Отправка данных пользователя: Иван Иванов...
Сервер ответил. Новая запись: {'name': 'Иван Иванов', 'username': 'Bret', ..., 'id': 11}
```

## Задача 3: Массовая загрузка из TXT (Читаем файл -> Конкурентные POST)

Менеджер скинул вам простой текстовый файл со списком задач на день (каждая задача с новой строки). Вам нужно написать скрипт, который прочитает этот файл и быстро (одновременно) создаст все эти задачи в системе через API.

**Условия**
- Создайте вручную файл `tasks.txt` и напишите там 3-4 любые строчки (например: `Купить молоко`, `Выучить aiohttp`, `Написать парсер`).
- Напишите корутину `bulk_create_tasks()`.
- Откройте файл `tasks.txt` и прочитайте все строки с помощью метода `.readlines()`. (Не забудьте очистить строки от символа переноса `\n` с помощью `.strip()`).
- В цикле сформируйте список задач (`tasks = []`). Каждая задача - это корутина вызова POST-запроса к `https://jsonplaceholder.typicode.com/todos`. Тело запроса: `{"title": имя_задачи, "completed": False, "userId": 1}`.
- Запустите все запросы одновременно с помощью `asyncio.gather()`.
- Распечатайте статусы ответов для каждой созданной задачи.

**Пример вывода:**
```text
Чтение файла tasks.txt... Найдено 3 задач.
Массовая отправка запросов...
Задача 'Купить молоко' создана с ID 201
Задача 'Выучить aiohttp' создана с ID 201
Задача 'Написать парсер' создана с ID 201
```
